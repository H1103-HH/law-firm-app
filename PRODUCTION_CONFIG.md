# 生产环境配置说明

## 问题说明

小程序上线后，全球合伙人页面不显示的原因通常是 **后端 API 域名未配置**。

在开发环境中，使用了 `localhost:3000` 作为后端地址，但小程序上线后必须配置正式的生产环境域名。

## 解决方案

### 方案一：创建 .env.local 配置文件（推荐）

在项目根目录创建 `.env.local` 文件，添加以下内容：

```bash
# 后端 API 地址（生产环境）
# 替换为你的实际后端域名
PROJECT_DOMAIN=https://your-backend-api.com

# 如果后端域名不需要 HTTPS，使用 http
# PROJECT_DOMAIN=http://your-backend-api.com
```

**注意**：
- 不要在 `.env.local` 中包含端口号（除非后端使用了非标准端口）
- 不要在域名后面加 `/` 斜杠
- 小程序上线后，必须使用 HTTPS 协议（小程序要求）

### 方案二：修改代码直接配置域名

如果不想使用环境变量，可以直接修改 `src/network.ts`：

```typescript
import Taro from '@tarojs/taro'

export namespace Network {
    const createUrl = (url: string): string => {
        if (url.startsWith('http://') || url.startsWith('https://')) {
            return url
        }
        // 直接配置生产环境域名
        const PROD_DOMAIN = 'https://your-backend-api.com'
        return `${PROD_DOMAIN}${url}`
    }

    // ... 其他代码保持不变
}
```

## 微信小程序域名配置

小程序上线后，除了代码中配置域名外，还需要在微信公众平台配置服务器域名白名单：

### 配置步骤

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入"开发" -> "开发管理" -> "开发设置"
3. 找到"服务器域名"部分
4. 在"request合法域名"中添加你的后端 API 域名
   - 例如：`https://your-backend-api.com`
5. 点击保存

**注意事项**：
- 域名必须是 HTTPS 协议
- 域名必须是经过 ICP 备案的
- 不能配置 IP 地址，必须是域名
- 不能配置端口号
- 不能包含子目录路径

### 域名要求

- ✅ 正确：`https://api.example.com`
- ✅ 正确：`https://www.example.com/api`
- ❌ 错误：`http://api.example.com`（必须 HTTPS）
- ❌ 错误：`https://192.168.1.1`（不能是 IP）
- ❌ 错误：`https://api.example.com:3000`（不能有端口）
- ❌ 错误：`api.example.com`（必须包含协议）

## 验证方法

### 1. 开发环境验证

在开发环境中，可以临时关闭"不校验合法域名"选项来测试：

1. 打开微信开发者工具
2. 点击右上角"详情"
3. 找到"本地设置"
4. 勾选"不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"

### 2. 生产环境验证

配置完成后，重新编译并上传小程序：

```bash
# 构建生产版本
pnpm build:weapp

# 上传小程序到微信后台
# 在微信开发者工具中点击"上传"按钮
```

### 3. 检查网络请求

在手机上打开小程序，使用调试工具查看网络请求：

1. 在小程序中打开右上角"..."
2. 选择"打开调试"
3. 返回小程序，查看网络请求是否正常

## 常见问题

### 问题1：request:fail url not in domain list

**原因**：域名未在微信公众平台的"request合法域名"中配置

**解决**：按照上述"微信小程序域名配置"步骤添加域名

### 问题2：request:fail 需要 HTTPS

**原因**：域名使用了 HTTP 协议，小程序要求 HTTPS

**解决**：为后端服务配置 SSL 证书，使用 HTTPS

### 问题3：request:fail 未能完成 HTTP 请求

**原因**：可能是以下原因之一：
- 后端服务未启动
- 域名配置错误
- 网络不通
- 后端接口不存在

**解决**：
- 检查后端服务是否正常运行
- 检查域名是否正确
- 在浏览器中直接访问 API 地址测试

### 问题4：接口返回 404

**原因**：API 路径配置错误

**解决**：检查代码中的 API 路径是否正确，例如：
- `/api/lawyers` 对应后端路由 `@Controller('lawyers')`
- 注意不要重复添加 `/api` 前缀（后端已全局配置）

## 后端部署要求

后端服务需要部署到生产环境，并满足以下要求：

1. **HTTPS 支持**：必须配置 SSL 证书
2. **ICP 备案**：域名必须经过 ICP 备案
3. **公网访问**：服务必须可以从公网访问
4. **稳定运行**：确保服务稳定，避免频繁宕机

## 推荐配置流程

1. **部署后端服务**
   - 使用云服务器（阿里云、腾讯云等）
   - 配置域名解析
   - 安装 SSL 证书

2. **配置小程序代码**
   - 创建 `.env.local` 文件
   - 配置后端域名

3. **配置微信后台**
   - 添加 request 合法域名

4. **测试验证**
   - 重新编译小程序
   - 在开发工具中测试
   - 上传体验版测试
   - 确认无误后发布
