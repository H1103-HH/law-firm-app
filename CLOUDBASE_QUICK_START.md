# 🚀 微信云开发快速部署指南

本文档提供最简化的部署步骤，按照顺序执行即可。

---

## 📋 准备工作

- ✅ 已安装微信开发者工具
- ✅ 已创建小程序项目
- ✅ 已克隆 GitHub 仓库到本地

---

## 🎯 部署步骤

### 第 1 步：开通微信云开发（5 分钟）

1. 打开微信开发者工具
2. 点击顶部导航栏的 **云开发** 按钮
3. 如果没有开通，点击 **开通**
4. 选择环境：
   - **环境名称**：`law-firm-app`
   - **基础版**（免费，每天 2GB 数据库 + 5GB 存储）
5. 点击 **开通**
6. 等待开通完成（通常 10-30 秒）

### 第 2 步：获取环境 ID（1 分钟）

1. 在云开发控制台顶部，可以看到 **环境 ID**
2. 复制环境 ID，格式类似：`law-firm-app-xxxx`
3. **保存这个 ID，后面会用到！**

---

### 第 3 步：创建数据库集合（10 分钟）

在云开发控制台，点击 **数据库**，然后依次创建以下集合：

#### 创建集合（6 个）

点击 **+** 按钮，依次创建：

1. **lawyers** - 律师信息
2. **users** - 用户信息
3. **consultations** - 咨询记录
4. **admin** - 管理员
5. **viewed_lawyers** - 浏览记录
6. **saved_cards** - 收集名片

#### 设置权限

对于每个集合，点击 **权限设置**，配置如下：

**lawyers** 权限：
```
所有用户可读，仅创建者可写
```

**users** 权限：
```
仅创建者可读写
```

**consultations** 权限：
```
所有用户可读，所有用户可写
```

**admin** 权限：
```
所有用户可读，仅创建者可写
```

**viewed_lawyers** 权限：
```
仅创建者可读写
```

**saved_cards** 权限：
```
仅创建者可读写
```

---

### 第 4 步：迁移数据（5 分钟）

#### 方式 1：使用迁移脚本（推荐）

在项目根目录运行：

```bash
# 在项目根目录
node scripts/migrate-to-cloudbase.js
```

脚本会生成 `migrations/` 目录下的 JSON 文件。

#### 方式 2：手动导出 Supabase 数据

如果脚本运行失败，可以手动操作：

1. 登录 Supabase Dashboard
2. 进入 **Table Editor**
3. 对于每个表，点击 **Export** → **CSV**
4. 转换为 JSON 格式

---

### 第 5 步：导入数据到微信云数据库（10 分钟）

对于每个集合：

1. 在云开发控制台，选择对应的集合（如 `lawyers`）
2. 点击 **导入**
3. 选择 `migrations/lawyers.json` 文件
4. 点击 **确定**
5. 等待导入完成
6. 重复以上步骤，导入所有 6 个集合

**导入顺序**：
1. lawyers
2. admin
3. users
4. consultations
5. viewed_lawyers
6. saved_cards

---

### 第 6 步：部署云函数（30 分钟）

#### 创建云函数

1. 在微信开发者工具中，打开项目
2. 右键点击 **cloudfunctions** 文件夹
3. 选择 **新建 Node.js 云函数**
4. 输入云函数名称（如 `getLawyers`）
5. 点击 **确定**

#### 复制云函数代码

对于每个云函数：

1. 在文件管理器中，找到 `cloudfunctions/getLawyers/index.js`
2. 复制文件内容
3. 在微信开发者工具中，打开 `cloudfunctions/getLawyers/index.js`
4. 粘贴代码
5. 保存文件（Ctrl+S）

#### 安装依赖

每个云函数都需要安装依赖：

1. 在微信开发者工具中，右键点击云函数目录（如 `getLawyers`）
2. 选择 **在终端中打开**
3. 运行 `npm install`
4. 等待安装完成

**注意**：对于简单的云函数（如 `getLawyers`），可以跳过 `npm install`，因为只使用了 `wx-server-sdk`。

#### 上传并部署

1. 在微信开发者工具中，右键点击云函数目录
2. 选择 **上传并部署：云端安装依赖**
3. 等待部署完成（通常 10-30 秒）

#### 需要部署的云函数（16 个）

按顺序部署以下云函数：

1. ✅ getLawyers
2. ✅ getLawyerDetail
3. ✅ login
4. ✅ createConsultation
5. ✅ getConsultations
6. ✅ replyConsultation
7. ✅ adminLogin
8. ✅ getAllLawyers
9. ✅ addLawyer
10. ✅ updateLawyer
11. ✅ deleteLawyer
12. ✅ addViewedLawyer
13. ✅ getViewedLawyers
14. ✅ saveCard
15. ✅ getSavedCards
16. ✅ removeCard

**提示**：可以批量复制多个云函数的代码，然后依次上传部署。

---

### 第 7 步：测试云函数（10 分钟）

#### 测试 getLawyers 云函数

1. 在微信开发者工具中，打开 **云开发** → **云函数**
2. 点击 `getLawyers` 云函数
3. 点击 **测试** 按钮

同样方式测试其他云函数，确保都能正常返回数据。

## 🆘 常见问题

### Q: 云函数部署失败怎么办？

A: 检查以下几点：
1. 云函数代码是否正确复制
2. 是否有语法错误
3. 查看部署日志，定位错误

### Q: 数据导入失败怎么办？

A: 检查以下几点：
1. JSON 文件格式是否正确
2. 字段是否匹配
3. 集合权限是否正确配置

### Q: 云函数调用超时怎么办？

A: 检查以下几点：
1. 环境变量 `env` 是否正确（不需要配置，使用默认值）
2. 数据库集合是否存在
3. 云函数代码是否有逻辑错误

### Q: 如何查看云函数日志？

A: 在微信开发者工具中：
1. 打开 **云开发** → **云函数**
2. 点击云函数
3. 点击 **日志**
